## Purpose

Defines how ArchLens packages are versioned, released, and published.

---

## Versioning

**Semantic Versioning (SemVer)** for all packages:

- **Major**: Breaking changes to public API or scoring methodology.
- **Minor**: New features, new rules, new metrics (backward compatible).
- **Patch**: Bug fixes, documentation, internal refactoring.

### Pre-1.0

During `0.x.y` development:

- Minor version bumps may include breaking changes.
- This is standard SemVer behavior for pre-1.0 releases.
- Goal: reach `1.0.0` when the scoring methodology and public APIs are stable.

---

## Release Tool

**Changesets** — manages versioning and changelog generation in monorepos.

### Workflow

1. Developer creates a changeset file describing the change:

    ```
    pnpm changeset
    ```

    This creates a markdown file in `.changeset/` describing the change and its version bump type.

2. Changeset files accumulate on `main` as PRs are merged.

3. When ready to release, Changesets opens a "Version Packages" PR that:
    - Bumps versions in `package.json` files.
    - Updates `CHANGELOG.md` in each changed package.
    - Removes consumed changeset files.

4. Merging the Version Packages PR triggers the release workflow:
    - Builds all packages.
    - Publishes changed packages to npm.
    - Creates GitHub release with aggregated changelog.

---

## Release Types

| Type     | Trigger                  | Scope                                        |
| -------- | ------------------------ | -------------------------------------------- |
| Stable   | Merge version PR to main | Published to npm with `latest` tag           |
| Canary   | Manual workflow dispatch | Published with `canary` tag from any branch  |
| Snapshot | Development only         | `pnpm pack` for local testing, not published |

---

## Package Publishing

All packages are published under the `@archlens` npm scope:

```
@archlens/types
@archlens/shared
@archlens/scanner
@archlens/parser
@archlens/graph
@archlens/analyzer
@archlens/rules
@archlens/scoring
@archlens/reporting
@archlens/cli        → also publishes bin: archlens
```

The `@archlens/cli` package is the primary user-facing package. Users install and use it via:

```bash
npx archlens analyze .
# or
pnpm add -g @archlens/cli
archlens analyze .
```

## Changelog

Each package maintains its own `CHANGELOG.md` generated by Changesets.

Changelog format:

```markdown
# @archlens/analyzer

## 0.2.0

### Minor Changes

- Added betweenness centrality metric computation

## 0.1.1

### Patch Changes

- Fixed instability index calculation when fan-in + fan-out = 0
```
